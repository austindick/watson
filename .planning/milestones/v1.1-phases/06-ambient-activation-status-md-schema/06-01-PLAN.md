```markdown
## Activation

**Entry point: blueprint gate**

1. Check for `blueprint/` directory in the current prototype directory
   - Not present -> **Stay silent.** Not a Watson prototype. Default Claude handles the message.

2. Check if message is pure coding/git/config with no prototype design intent (Tier 0)
   - Signals: bash/git commands, TypeScript errors, linting, test runs, file renames, dependency installs — anything with no prototype design context
   - YES -> **Stay silent.** Defer to default Claude.

3. Check for `blueprint/STATUS.md`
   - **Exists -> Returning prototype.** Parse YAML frontmatter. Display summary:
     "[prototype_name] — [N] section(s) built ([list]). Last discussed: [last_discussed]."
     Then offer choices: "Continue building / Discuss changes / Start fresh"
     Wait for user selection, then proceed to Intent Classification.
   - **Not present -> New or incomplete prototype.**
     Was `/watson` explicitly invoked?
     - YES -> Run Setup Flow, then proceed to Intent Classification.
     - NO -> **Stay silent.** Intentional: don't scaffold without explicit request.
```

Target: ~20 lines for this section. The activation logic is a decision tree — blueprint gate first, then Tier 0, then STATUS.md check.
