This task runs the Watson pipeline agents against a real Figma frame to verify they work end-to-end.

**Setup:**
1. Ask the user for a Figma frame URL with sufficient complexity (auto-layout, mixed spacing, design-system-matchable + unmapped elements)
2. Create a temporary test prototype directory structure:
   ```
   /tmp/watson-smoke-test/
     blueprint/
     .watson/
       sections/
   ```
3. Resolve libraryPaths from the Watson design-system book. Read LIBRARY.md and BOOK.md to select relevant chapters. For layout agent: provide the global-theme chapter path (contains spacing/radius tokens). For design agent: provide components chapter + global-theme chapter paths.

**Decomposer run:**
4. Simulate decomposer dispatch: pass the Figma URL to the decomposer agent logic. Record the sections[] JSON output. Pick ONE section for the parallel test.

**Parallel dispatch timing test (addresses bug #7406):**
5. Record wall-clock start time
6. Dispatch layout agent (background) with:
   - nodeId: from decomposer output for the chosen section
   - sectionName: from decomposer output
   - libraryPaths: [path to global-theme CHAPTER.md]
   - blueprintPath: /tmp/watson-smoke-test/blueprint/
7. Dispatch design agent (background) SIMULTANEOUSLY with:
   - nodeId: same section nodeId
   - sectionName: same section name
   - libraryPaths: [path to components CHAPTER.md, path to global-theme CHAPTER.md]
   - blueprintPath: /tmp/watson-smoke-test/blueprint/
8. Wait for both to complete
9. Record wall-clock end time
10. Calculate: if total time > 1.5x the longer of the two individual agent times, flag as SERIALIZED (bug #7406 triggered)

**Schema compliance check:**
11. Read the output LAYOUT.md at .watson/sections/{sectionName}/LAYOUT.md
    - Verify headers: "## Token Quick-Reference", "## Component Tree", "## Annotated CSS"

    - Verify no bare px values without token variable reference
    - Verify token names come from library (not hardcoded --faux-*)
12. Read the output DESIGN.md at .watson/sections/{sectionName}/DESIGN.md
    - Verify headers: "## Component Mapping", "## Typography", "## Color Tokens", "## Unmapped Values"
    - Verify line count < 80
    - Verify Unmapped Values section present (even if _None_)
    - Verify component names come from library (not hardcoded FauxDS names)

**Report timing and results to conversation.**
