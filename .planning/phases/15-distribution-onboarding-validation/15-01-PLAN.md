**NOTE on install flow:** CONTEXT.md originally specified "one command: claude plugin add austindick/watson". **Structure (per CONTEXT.md locked decisions):**

1. **Beta badge** at top: `> **Beta** — internal Faire tool. DM Austin in Slack with questions.`

2. **One-sentence description:** Watson helps you design and prototype UI components in the Faire Prototype Playground using Claude Code.

3. **Prerequisites section:**
   - Claude Code installed and working
   - Figma MCP configured (see setup section below)

4. **Install section** — exact 3-step command sequence:
   ```
   /plugin marketplace add austindick/watson
   /plugin install watson@watson
   /reload-plugins
   ```
   Then the manual ambient rule copy step (DIST-04):
   - After install, copy the ambient rule file so Watson activates automatically in the Playground:
   ```bash
   cp ~/.claude/plugins/cache/watson/watson/*/skills/watson/references/watson-ambient.md ~/.claude/rules/watson-ambient.md
   ```
   Note: the glob `*` handles any version in the cache path.

5. **Figma MCP Setup section** — inline 3-step instructions (not a link):
   - Step 1: Install the Figma MCP server (link to Figma's MCP plugin page)
   - Step 2: Add Figma MCP to Claude Code settings (add to mcpServers in settings)
   - Step 3: Verify: ask Claude "can you access Figma?" — it should confirm the connection

6. **Auto-Update section** (optional, brief):
   - To receive future Watson updates automatically:
   ```
   /plugin -> Marketplaces tab -> select "watson" -> Enable auto-update
   ```
   - Or update manually anytime: `/plugin marketplace update watson`

7. **Versioning note (DIST-02):**
   - Watson version bumps on every push to main. Patch for fixes, minor for features.
   - No action needed from teammates — updates apply automatically if auto-update is enabled, or on next `/plugin marketplace update watson`.
   - This section establishes the version-bump discipline required for DIST-02 auto-update to function.

8. **Troubleshooting section** — 2-3 items (Claude's discretion on exact wording):
   - `/watson` not found after install: Run `/reload-plugins`. If still missing, restart Claude Code.
   - Watson doesn't suggest itself in Playground: Check that `~/.claude/rules/watson-ambient.md` exists. Re-copy from install step if missing.
   - Figma MCP errors: Verify MCP server is running. Try "can you access Figma?" to diagnose.

**Constraints:**
- No architecture details, no agent explanations, no file tree (per CONTEXT.md)
- No CHANGELOG (deferred idea)
- Total length: aim for 60-90 lines of markdown
- Use simple language, numbered steps, code blocks for commands
