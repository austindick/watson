  <action>
Extend the existing `agents/librarian.md` (created in 02-01 with generate mode) to add update mode and a LIBRARY.md auto-update step that runs at the end of both modes.

**Update Mode Section — add after the existing Generate Mode section:**

The update mode instructions should follow the pattern from Loupe's `utilities/librarian.md` but adapted for Watson's Book/Chapter/Page hierarchy:

1. **Load existing book:** Read `{outputBookPath}/BOOK.md` frontmatter. Parse chapter manifest. Read each CHAPTER.md (and PAGE.md if chapters use pages). Build an in-memory map of all content keyed by chapter ID and page ID.

2. **Re-scan source:** Run the same scan logic as generate mode on `sourcePaths[]`. Store results in memory — do not write anything yet.

3. **Compute source_hash:** Generate sha256 hash of the concatenated source file contents. Compare against the existing BOOK.md `source_hash` frontmatter field. If hashes match, print "No source changes detected. Book is up to date." and skip to LIBRARY.md auto-update step.

4. **Diff at chapter and page level:**
   - New chapters/pages: entries in scan not present in existing book
   - Modified chapters/pages: entries where content differs from existing
   - Removed chapters/pages: entries in existing book not found in scan

5. **Apply changes surgically:**
   - **New:** Create new CHAPTER.md or PAGE.md files in the correct directory. Add to BOOK.md chapter manifest.
   - **Modified:** Overwrite only the changed CHAPTER.md or PAGE.md file. Preserve unchanged files verbatim.
   - **Removed:** Do NOT delete. Add `[DEPRECATED]` to the chapter/page title. Keep in BOOK.md manifest with a `deprecated: true` field in the chapter entry.

6. **Update BOOK.md metadata:** Update `last_updated` to current ISO date. Update `source_hash` to the new hash. Update chapter manifest if chapters were added or deprecated.

**LIBRARY.md Auto-Update Step — add as the FINAL step in BOTH generate and update modes:**

After writing/updating the book, the Librarian MUST update `{libraryPath}/LIBRARY.md`:

1. Read the current LIBRARY.md
2. For the book that was just generated/updated:
   - If the book entry already exists in LIBRARY.md: update its metadata (last_updated, chapter listing, chapter count)
   - If the book is new: add a new entry under `## Books`
3. Update the top-level metadata: `**Last updated:**` to current date, `**Books:**` to current count
4. Each book entry in LIBRARY.md must include:
   - `### {Book Title}` heading
   - `**Path:** library/{book-dir}/BOOK.md`
   - `**Last updated:** {date}`
   - `**Chapters:** {comma-separated chapter names with counts where relevant}`
   - `**Use when:** {copied from BOOK.md use_when field}`

**Critical constraints to include in the agent file:**
- Update mode is surgical: unchanged chapters/pages are never rewritten
- source_hash comparison is the fast-path: if source hasn't changed, skip the full diff
- LIBRARY.md auto-update is mandatory — it runs at the end of EVERY Librarian operation, not just generate
- The Librarian is a background agent — no interactive prompts. All inputs come via parameters.
- Agent file must reference `@references/book-schema.md` for output format conformance

**File size budget:** The complete librarian.md (generate + update + LIBRARY.md auto-update) should target ~2,000 tokens. Use bullet lists, not prose. Front-load constraints.
