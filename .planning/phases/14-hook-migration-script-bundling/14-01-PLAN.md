**scripts/watson-session-start.js** — Three responsibilities in order:
1. **Recovery notification**: Check if `/tmp/watson-active.json` exists. If yes, log the recovery message (same emoji + text as current bash one-liner).
2. **Ambient rule auto-copy**: Use `__dirname` to derive source path: `path.join(__dirname, '..', 'skills', 'watson', 'references', 'watson-ambient.md')`. Target: `~/.claude/rules/watson-ambient.md`. Copy ONLY if target does not exist AND source exists. Log confirmation on copy, silent skip if already present.
3. **StatusLine auto-write**: Read `~/.claude/settings.json` with safe read-modify-write pattern. Only write statusLine if: (a) missing entirely, OR (b) currently points to `share-proto-statusline.js`. Set to `{ type: "command", command: "node \"<plugin_root>/scripts/watson-statusline.js\"" }` where plugin_root is derived from `path.resolve(__dirname, '..')`. If statusLine already points to `watson-statusline.js` or any other custom script, skip. Handle JSON parse errors by logging error and aborting (do NOT silently overwrite corrupt settings). Handle ENOENT by starting with empty object.

**scripts/watson-session-end.js** — Port of the current inline Node one-liner to a proper script:
1. Check if `/tmp/watson-active.json` exists. If not, exit 0.
2. Read and parse the file.
3. If `branch` field present, write `/tmp/watson-session-end.json` with `{branch, actions: s.actions || [], timestamp: new Date().toISOString()}`.
4. Delete `/tmp/watson-active.json` (unlink in try/catch).
5. Wrap entire logic in try/catch — on any error, still attempt to unlink the active file.

**scripts/watson-statusline.js** — Fork from `~/.claude/hooks/share-proto-statusline.js`:
1. Copy the entire 117-line file.
2. REMOVE lines 75-87 (the standalone dev server block: the `if (!proto)` block that reads `/tmp/share-proto-dev-${process.ppid}.json`).
3. KEEP everything else: OSC 8 link helper, portOpen, stdin parsing, context bar, share-proto tunnel links (reads `/tmp/share-proto.json`), git branch, Watson active indicator, stdout.write.
4. Do NOT modify any remaining logic — this is a surgical removal, not a rewrite.
