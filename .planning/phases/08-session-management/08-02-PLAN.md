**Part A: Extend SessionEnd hook in settings.json**

Find the SessionEnd hook (line ~90-95) that currently has:
```json
"command": "rm -f /tmp/watson-active.json"
```

Replace with a Node.js one-liner that preserves actions for next-session recovery before deleting the state file:

```
node -e "const fs=require('fs'); const f='/tmp/watson-active.json'; if(!fs.existsSync(f)) process.exit(0); try { const s=JSON.parse(fs.readFileSync(f,'utf8')); if(s.branch) fs.writeFileSync('/tmp/watson-session-end.json', JSON.stringify({branch:s.branch,actions:s.actions||[],timestamp:new Date().toISOString()})); fs.unlinkSync(f); } catch(e) { try { fs.unlinkSync(f); } catch(e2) {} }"
```

This writes `/tmp/watson-session-end.json` with `{branch, actions, timestamp}` before deleting the state file. On next Watson activation, the session recovery check (added by Plan 01 in SKILL.md) reads this file, compiles actions into a summary, writes the session entry to STATUS.md, and deletes the temp file.

**Part B: Extend /watson off handler in SKILL.md**

Find the `/watson off` handler (currently at line 19 or wherever Plan 01 placed it). It currently deletes the state file. Insert session entry write BEFORE the state file deletion:

```
1. Read /tmp/watson-active.json
2. If `branch` and `actions` fields exist:
   a. Get current user: `git config --get user.name` (or use STATUS.md owner_github if available)
   b. Compile actions array into summary: join with ", ", truncate at 80 chars
   c. Read STATUS.md `sessions:` array from the current branch's blueprint/STATUS.md
   d. Prepend new entry: {timestamp: current ISO string, summary: compiled summary, who: username}
   e. If sessions count >= 10, drop the last (oldest) entry
   f. Update STATUS.md `sessions:` via Edit tool
      - If `sessions: []` (empty compact), replace with block sequence format
      - If sessions already has entries, prepend the new entry
3. Delete /tmp/watson-active.json
4. Respond "Watson deactivated for this session."
```
