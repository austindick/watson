**Part A: Create `~/.claude/rules/watson-ambient.md`**

First, create the `~/.claude/rules/` directory if it does not exist:
```bash
mkdir -p ~/.claude/rules
```

Then create the rule file with this exact content:

```markdown
---
paths:
  - "src/pages/**"
---

# Watson Ambient Activation

When working with files under `src/pages/**`, follow the Watson skill's Activation section.

Watson is a design prototyping assistant. If a `blueprint/` directory exists in the prototype directory, Watson should handle design-related messages. Check the Watson skill (`/watson`) Activation section for the full decision tree: blueprint gate, Tier 0 passthrough, STATUS.md routing.

**Key rules:**
- If `blueprint/` does NOT exist in the prototype directory, stay silent — this is not a Watson prototype.
- If the message is pure coding/git/config (no design intent), stay silent — defer to default Claude behavior.
- If `blueprint/STATUS.md` exists, this is a returning prototype — follow Watson's returning flow.
- If no `blueprint/STATUS.md` and no explicit `/watson` invocation, stay silent — don't scaffold without explicit request.

When Watson should engage, load and follow the full Watson skill at `~/.claude/skills/watson/SKILL.md`.
```

This rule activates on any file read under `src/pages/**` and instructs Claude to check for and follow the Watson skill's Activation section. The `paths:` field here is on the RULE, not on the SKILL — so `/watson` slash command registration is unaffected.
